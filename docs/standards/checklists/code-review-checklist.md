## 功能正确性

- [ ] 功能是否符合设计文档要求？
- [ ] 边界条件是否处理（参数为 0、最大值、NULL）？
- [ ] 是否有明显的逻辑错误？
- [ ] 是否考虑了异常情况？

---

## MISRA-C 合规

- [ ] 是否使用了裸 int/unsigned？（应使用 stdint.h）
- [ ] 是否有隐式类型转换？（应显式 cast）
- [ ] 是否有 goto 语句？（禁止）
- [ ] 是否有递归？（禁止）
- [ ] switch 是否有 default 分支？（必须）
- [ ] 指针参数若只读是否声明为 const？（规则 8.13）

---

## 编码规范

- [ ] 函数命名是否符合规范（模块前缀 + 动词）？
- [ ] 变量命名是否清晰？
- [ ] 函数长度是否 ≤ 60 行？
- [ ] 是否有过深的嵌套（> 4 层）？
- [ ] 文件头注释是否完整？

---

## 测试

- [ ] 是否有单元测试？
- [ ] 测试是否覆盖主要场景（正常、边界、异常）？
- [ ] 测试是否通过？

---

## 文档

- [ ] 公开 API 是否有注释说明？
- [ ] 复杂逻辑是否有注释？
- [ ] 是否需要更新设计文档？
